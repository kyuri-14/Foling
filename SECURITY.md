# セキュリティポリシー

Foling のセキュリティ上の問題に関する報告方法と、既知のセキュリティ特性をまとめます。

## サポート対象バージョン

セキュリティ修正は最新リリースおよび `main` ブランチに対して行います。

| バージョン | サポート |
| --- | --- |
| 最新リリース (`0.10.x`) | ✅ |
| `main` (開発版) | ✅ |
| それ以前 | ❌ |

## 脆弱性の報告

**公開 Issue には投稿しないでください。** 脆弱性は非公開でお知らせください。

- **推奨: GitHub の「Security」→「Report a vulnerability」**（プライベート脆弱性報告）。
  リポジトリの Settings → Code security で *Private vulnerability reporting* を有効化すると使えます。
- メールでの受付を希望する場合は、この行に連絡先アドレスを追記してください（任意）。

報告には、再現手順・影響範囲・対象バージョン/OS・（可能なら）PoC を含めてください。

### 対応の流れ

- 受領から **7 日以内**を目安に一次返信します。
- 妥当性を確認のうえ修正を作成し、リリースします。
- 調整のうえで公開開示（必要に応じて CVE 採番・クレジット掲載）します。
- 報告者のプライバシーを尊重します。

## 既知のセキュリティ特性

Foling はローカルのデスクトップアプリです。設計上の前提を明記します。

- **コード署名なし**（現時点）: 配布バイナリは未署名のため、Windows SmartScreen /
  macOS Gatekeeper が警告を出すことがあります。正式配布時に証明書で署名予定です。
- **プラグインは任意の JavaScript を実行します**: exporter プラグインは Web Worker 上で動作し、
  初回実行時に同意を求めますが、**完全なサンドボックスではありません**。信頼できる提供元の
  プラグインのみ使用してください。
- **プレビューサーバはローカル限定**: `127.0.0.1` のランダムポートで待ち受け、外部公開しません。
- **Content-Security-Policy**: `tauri.conf.json` の `security.csp` で self + ローカルプレビューに
  限定しています。
- **ファイルアクセス権限は最小限**: ダイアログ（`dialog:allow-open` / `allow-save`）と
  プロジェクトフォルダ配下の読み書きに限定しています。
